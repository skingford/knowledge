## 实施计划

### Phase 1：补充核心技术栈的标准答案
为每个"必须能讲清"的知识点补充 **2-4 句口述级答案**：

1. **Go 调度模型**：补充 GMP 完整口述答案、syscall 阻塞机制、抢占式调度原理
2. **Go 内存管理**：补充三色标记具体流程、写屏障类型、逃逸分析判断规则
3. **Go 并发控制**：补充 channel 内部结构、context 取消树传播机制、Mutex 饥饿模式细节
4. **MySQL**：补充 B+Tree 为什么适合、MVCC 工作流程、深分页优化方案对比
5. **Redis**：补充穿透/击穿/雪崩的治理方案、RDB vs AOF 对比表、Cluster 槽位机制
6. **Kafka**：已有内容较完整，微调即可

### Phase 2：补充高频追问 Q1-Q7 的完整参考答案
将每个 Q 的"回答骨架"扩展为 **可直接口述的完整参考答案**（类似场景题的总结版答案格式）

### Phase 3：增加"考前速查卡片"章节
在文档末尾（推荐复习顺序之前）增加一个速查区域：
- 每个主题一张卡片，包含：核心结论 + 关键数字 + 一句话答案
- 覆盖：Go / MySQL / Redis / Kafka / AI / IM / K8s 七大主题
- 目标：考前 30 分钟快速过一遍

### Phase 4：增加"面试题 → 底层原理"对照表
用表格形式建立：面试题 → 考察的底层原理 → 一句话标准答案 的映射关系

### Phase 5：审查与微调
- 检查全文一致性、术语统一
- 优化快速导航目录
- 确保所有新增内容与现有三道场景题不重复
