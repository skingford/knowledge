# 7 天 Agent 学习路线

Day 1：理解 Agent 到底是什么
目标：分清 LLM、Bot、Workflow、Agent
学习内容：
什么是大模型
什么是 prompt
什么是 tool calling
Agent 和普通问答助手的区别
产出：
你能用自己的话解释：Agent = LLM + Tools + Memory + Workflow
Day 2：学会调用 LLM API
目标：把模型真正用起来
学习内容：
API key、request、response
system prompt / user prompt
温度、max tokens、结构化输出
实操：
写一个最简单的聊天脚本
推荐：
先用 Python 或 TypeScript 二选一
Day 3：学会 Tool Calling
目标：让模型不仅会说，还会“做事”
学习内容：
function calling
工具描述怎么写
模型如何选择调用工具
实操：
做一个能调用 天气、搜索、时间 三个工具的小助手
关键理解：
Agent 的核心不是会聊天，而是会在合适的时候调用能力
Day 4：学 Workflow，而不是一上来追求自治
目标：理解可控 Agent 的真正工程形态
学习内容：
step-by-step workflow
状态机
重试、失败处理、人工兜底
实操：
做一个固定流程：
理解问题 -> 判断是否查资料 -> 调工具 -> 汇总答案
Day 5：学 RAG 和 Memory
目标：让 Agent 能利用外部知识
学习内容：
embedding
chunking
retrieval
rerank
短期记忆 vs 长期记忆
实操：
给 Agent 接一个简单知识库，比如几篇 markdown 或 PDF
注意：
很多“智能”其实不是模型更强，而是检索做得更对
Day 6：学评测与安全
目标：别把 demo 当产品
学习内容：
如何评估答案质量
幻觉问题
prompt injection
工具权限边界
成本、延迟、稳定性
实操：
给你的 Agent 设计 10 个测试问题
记录成功率和失败原因
Day 7：做一个完整小项目
目标：把前 6 天的内容串起来
推荐项目三选一：
文档问答 Agent
联网搜索 Agent
个人办公 Agent（天气/日程/提醒）
最终要求：
能回答问题
能调用至少 2 个工具
有基本错误处理
有简单日志
最好能展示一个完整工作流
一周后你应该达到的水平
知道 Agent 不是“套壳聊天”
能独立做一个单 Agent + 多工具的小项目
理解为什么 workflow 比“完全自治”更适合落地
知道下一步该补 RAG / eval / observability / multi-agent
推荐学习资料搭配
官方文档：
OpenAI docs
Anthropic docs
LangGraph docs
视频/课程：
DeepLearning.AI 的 LLM / Agent 课程
Hugging Face 教程
Karpathy 的 LLM 相关视频
实战方向：
找 GitHub 上最小可运行 demo，自己改一遍比只看文章有效得多

## Agent 和 LLM，先懂模型，再懂怎么把模型接成可工作的系统。
LLM 入门先看这些：
Transformer 基础：注意力机制、token、上下文窗口
Prompting：zero-shot、few-shot、chain-of-thought 的使用边界
Function Calling / Structured Output：这是做 Agent 的关键
RAG：检索、切块、embedding、rerank
Fine-tuning 基础：先知道适用场景，不必一开始就训练
Agent 入门重点学这些：
工具调用：模型怎么决定何时调 API / 搜索 / 数据库 / 脚本
任务规划：把复杂任务拆成多步
Memory：短期上下文、长期记忆、用户画像
Workflow：固定流程、状态机、人工兜底
Multi-agent：最后再学，别开局就上
推荐资料方向：
OpenAI API docs：看 tool calling、structured outputs、responses
Anthropic docs：看 tool use、agent design、context engineering
LangChain / LangGraph docs：看 agent 与 workflow 编排
DeepLearning.AI 上的 LLM / Agent 课程
Hugging Face 的 NLP/Transformer 教程
Andrej Karpathy 的 LLM/Tokenizer/Transformer 相关公开视频
初学者最实用的学习顺序：
先懂 LLM 基础概念
再学 API 调用和 prompt
再学 function calling
再做一个带工具的单 Agent
再补 RAG、eval、memory
练手项目建议：
一个能联网搜索的问答助手
一个读取本地文档的 RAG 助手
一个能调用天气/地图/日历 API 的 Agent
一个简单的“自动拆任务并执行”的工作流 Agent
常见误区：
只学 prompt，不学系统设计
只追求多 Agent，不做单 Agent 落地
不做评测，靠感觉判断效果
把所有问题都交给模型，不做规则和边界控制

## 先学的核心知识
LLM 基础：token、上下文窗口、temperature、结构化输出
Prompt 基础：system prompt、few-shot、约束输出
Tool Calling：让模型会调用搜索、数据库、API、脚本
Workflow：把 Agent 做成可控流程，不是纯放养
RAG：让 Agent 结合外部知识工作
Eval：评估效果，而不是只看 demo

推荐学习框架
LangGraph：最适合工程化入门
LangChain：生态广，但别一开始学太杂
CrewAI / AutoGen：适合了解多 Agent，但先知道就好
如果你偏实战，也可以直接用各家原生 SDK 手写一个小 Agent，反而更容易理解本质
最适合练手的项目
搜索 Agent：用户提问 -> 联网搜索 -> 总结答案
RAG Agent：读取本地文档/PDF -> 检索 -> 回答
办公 Agent：天气、日历、提醒、消息发送
数据 Agent：查表、汇总、输出日报
建议学习顺序
先学 LLM API
再学 prompt 和结构化输出
再学 tool calling
再学 workflow
再学 RAG
最后看 multi-agent
常见误区
一上来就学 multi-agent
只会写 prompt，不会做流程控制
不做测试，不知道 Agent 为啥失败
把 Agent 当“万能自动人”，不给权限边界
