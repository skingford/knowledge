## 按阶段
初级阶段：打基础，先具备“看懂系统”的能力
学习目标：理解一个系统是怎么跑起来的，而不只是会写功能
重点内容：
计算机基础：操作系统、网络、数据结构、算法基础
数据库：MySQL、索引、事务、锁、SQL 优化
后端基础：接口设计、认证鉴权、缓存、消息队列入门
设计能力：面向对象设计、设计模式、模块拆分
Linux：常用命令、进程、内存、磁盘、网络排查
这个阶段要达到：
能独立设计一个中小型系统
能看懂典型 Web 架构
能分析慢 SQL、接口慢、服务异常这些常见问题
推荐实践：
做一个完整后台系统
加入 Redis、MQ、日志、监控
画出系统架构图和部署图
中级阶段：进入“系统设计”和“分布式”能力建设
学习目标：能设计可扩展、可维护、抗压的系统
重点内容：
高并发：缓存、限流、降级、熔断、异步削峰
高可用：主从、集群、故障转移、容灾、重试机制
分布式系统：CAP、BASE、一致性、分布式事务
微服务：服务拆分、注册发现、配置中心、网关
中间件：Redis、Kafka/RabbitMQ、ES、Nginx
可观测性：日志、指标、链路追踪、告警
这个阶段要达到：
能主导一个中大型系统模块设计
能根据业务增长做架构演进
能处理性能瓶颈和稳定性问题
推荐实践：
设计一个秒杀系统 / 订单系统 / 支付系统
做压测并分析瓶颈
输出容量评估、监控方案、容灾方案
高级阶段：从“技术实现”升级到“架构治理”和“业务抽象”
学习目标：从单点技术强，走向全局设计强
重点内容：
架构方法论：领域驱动设计 DDD、事件驱动、分层架构、整洁架构
业务架构：领域划分、边界上下文、系统拆分与整合
平台化：中台、共享能力平台、统一技术底座
治理体系：代码规范、发布流程、变更管理、SLA/SLO、风险治理
云原生：Docker、Kubernetes、服务网格、弹性伸缩
安全架构：权限体系、审计、数据安全、攻防意识
成本与效率：资源成本、研发效率、稳定性投入产出比
这个阶段要达到：
能从业务目标反推技术方案
能主导跨团队架构设计
能做系统演进规划，而不是只做局部优化
推荐实践：
主导老系统拆分与迁移方案
设计多团队共享平台
输出技术规划、架构评审模板、风险清单

## 初级阶段书单：建立扎实基本功
《图解 HTTP》：适合补网络与 Web 通信基础
《MySQL 必知必会》 或 《高性能 MySQL》：先入门再进阶
《Redis 设计与实现》：理解缓存和数据结构
《深入理解计算机系统》：建立系统底层认知
《UNIX/Linux 系统管理技术手册》 或常见 Linux 实战资料：补运维基础
《Head First 设计模式》 或 《设计模式》：建立设计思维
中级阶段书单：进入分布式与高并发
《大型网站技术架构》：很适合中文语境下入门架构演进
《从 0 开始学架构》：偏方法论，适合建立整体视角
《高并发系统设计 40 问》：适合结合业务场景理解问题
《数据密集型应用系统设计》：非常重要，建议重点读
《Kafka 权威指南》：理解消息系统
《Elasticsearch 实战》：搜索与检索系统能力补充
《凤凰项目》 / 《独角兽项目》：帮助理解工程协作与交付体系
高级阶段书单：从技术点到架构方法论
《软件架构设计：大型网站技术架构与业务架构融合之道》 或同类架构方法书
《实现领域驱动设计》：适合进入 DDD
《领域驱动设计》：经典但偏厚，建议中高级后再啃
《企业集成模式》：理解系统集成与消息模式
《Clean Architecture》：看架构分层与边界控制
《Building Evolutionary Architectures》：理解演进式架构
《Site Reliability Engineering》：稳定性治理非常值得看
