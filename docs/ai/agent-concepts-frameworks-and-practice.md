## 先搞懂 5 个核心概念

### 1. LLM

- 模型本身负责理解、生成、决策
- 需要知道它的上下文窗口、输出边界、成本和延迟

### 2. Tools

- Agent 用来调用外部能力的入口
- 例如搜索、代码执行、数据库、日历、消息发送

### 3. Memory

- 包括短期上下文和长期用户记忆
- 解决“当前会话连续性”和“历史信息复用”两类问题

### 4. Planning

- 把复杂任务拆成多步
- 让 Agent 从“单次回答”变成“分步骤执行”

### 5. Workflow

- 把步骤放进稳定可控的流程里
- 用于控制状态、路由、重试、失败兜底和人工接管

## 推荐优先看的框架与资料

### 官方文档

- OpenAI：重点看 Tool Calling、Structured Output、Agent Patterns
- Anthropic：重点看 Tool Use、Context Engineering、构建可靠 Agent

### 工作流编排

- LangGraph：非常适合学习工作流型 Agent

### 多 Agent 先了解即可

- CrewAI / AutoGen：适合了解多 Agent 协作
- 先不要陷太深，不然容易跳过最关键的单 Agent 基础

## 值得看的内容类型

- 官方 Quickstart
- Cookbook / Examples
- “How to build agents” 类技术博客
- GitHub 上可运行的最小 Demo

## 最适合入门的实战项目

### 问答 Agent

- 能调用搜索工具

### RAG Agent

- 能读 PDF 或文档知识库

### 办公 Agent

- 能查天气、日程、发消息

### 数据 Agent

- 能查表、汇总、输出报告

## 推荐学习顺序

1. 先学 Prompt + API
2. 再学 Tool Calling
3. 再学 Workflow / State Machine
4. 再加 Memory / RAG
5. 最后再看 Multi-agent

## 一个重要判断

真正能落地的 Agent，通常不是“超级自治体”。

更常见、也更可靠的形态是：

- LLM
- 工具
- 工作流
- 人工兜底

这比“完全放养式自治”更容易：

- 调试
- 评测
- 上线
- 控制风险
