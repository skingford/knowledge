# Agent 入门基础与学习顺序

## 适合人群

- 想系统入门 Agent 开发，但还没建立完整学习框架
- 已经会调用 LLM API，但不清楚下一步该补什么
- 容易陷入“框架很多、概念很多、项目太散”的初学者

## 学习目标

- 先补齐 Agent 入门最基础的能力块
- 理解 Agent 核心 4 件事和它们之间的关系
- 建立更稳的学习顺序，避免一开始就追求多 Agent 自治

## 第一阶段先补什么

### 1. 先学一门实现语言

- Python 或 TypeScript 二选一即可
- 不要一开始两门都学
- 优先选你后续最可能拿来写工具、脚本和 API 调用的语言

### 2. 补 HTTP / API 调用

- 理解请求、响应、鉴权、超时、重试
- 会调用 REST API 或 SDK
- 知道如何把外部能力接进自己的 Agent

### 3. 补 Prompt 基础和结构化输出

- 理解 System Prompt、User Prompt、Few-shot 的作用
- 知道如何约束输出格式
- 学会 JSON / Structured Output，减少模型自由发挥

## 再理解 Agent 核心 4 件事

### 1. 模型

- 模型负责理解输入、生成输出、决定是否调用能力
- 需要知道模型的上下文窗口、输出约束、成本和延迟边界

### 2. 工具调用

- Tool Calling / Function Calling 是 Agent 真正“能做事”的基础
- 工具让模型可以搜索、查天气、访问数据库、调用脚本

### 3. 记忆与上下文

- 短期上下文解决当前会话连续性
- 长期记忆解决用户偏好、历史事实、跨任务信息复用
- 很多所谓“记忆”问题，本质上是上下文管理问题

### 4. 工作流与状态机

- 工作流负责控制步骤、路由、重试、失败兜底
- 状态机让 Agent 从“会说”变成“可控执行”

## 为什么先学工作流型 Agent

- 入门最好先学工作流型 Agent
- 先做“调用模型 -> 调工具 -> 解析结果 -> 决策下一步”这类确定性流程
- 先把单 Agent + 多工具做稳定，再碰多代理自治

这样做的原因：

- 更容易调试
- 更容易加日志和失败兜底
- 更容易做评测
- 更接近真实工程落地

## 第一阶段推荐资料方向

### 官方文档

- OpenAI：重点看 Function Calling、Structured Output
- Anthropic：重点看 Tool Use

### 编排与状态

- LangGraph：非常适合理解有状态 Agent 和节点编排

### Prompt 优化思路

- DSPy：适合理解“把 Prompt 当程序优化”

### 多 Agent 了解即可

- AutoGen / CrewAI：适合了解多 Agent 协作
- 不建议一上来就重度依赖

## 第二阶段要补的工程能力

### RAG 基础

- 切块
- 召回
- Rerank
- 上下文拼接

### 评测 / Evals

- 正确率
- 稳定性
- 幻觉率
- 工具成功率

### 可观测性

- 日志
- Trace
- Token / 成本统计

### 安全

- Prompt Injection
- 工具权限边界
- 数据泄露风险

## 最值得练的 5 个小项目

- 文档问答助手
- 会查天气、日历、网页的工具型助手
- 能读数据库并生成报表的分析 Agent
- 一个带长期记忆的个人助理
- 一个能自己分解任务并调用脚本执行的 Coding / Task Agent

## 推荐学习顺序

1. 先做单 Agent + 2 到 3 个工具
2. 再加 RAG
3. 再做状态机 / 工作流
4. 最后再碰多 Agent

## 常见误区

- 一开始就追求“全自动自治”
- 不做 Eval，只看 Demo 效果
- 把 Prompt 堆很长，当成系统设计
- 工具权限不给边界
- 没有失败兜底和人工接管
