# 向量数据库选型：pgvector、Milvus、Weaviate、Qdrant

## 适合人群

- 正在做 RAG 或知识库系统，不确定一开始该选哪种向量存储的工程师
- 已经有 PostgreSQL，但又担心以后规模起来不够的人
- 想从工程成本、运维复杂度和演进路径角度看选型的人

## 学习目标

- 理解向量数据库选型不是“谁更先进”，而是“谁更适合当前阶段”
- 建立 pgvector、Milvus、Weaviate、Qdrant 的典型适用边界
- 学会按数据规模、团队能力、检索复杂度和运维成本做判断

## 先别把选型问题想得太神秘

很多团队做 RAG 时，第一周就开始讨论：

- 要不要一上来就上专门向量数据库
- 谁的 ANN 更强
- 哪家以后更容易做混合检索

这些问题都重要，但排序往往太靠前了。

对大多数团队来说，先要回答的是：

- 你现在有没有现成数据库基础设施
- 数据量到底多大
- 检索是不是业务核心瓶颈
- 团队愿不愿意多维护一套新系统

## 先看 4 个核心判断维度

| 维度 | 你要问的问题 |
| --- | --- |
| **规模** | 现在和未来 6-12 个月的向量规模会到什么量级 |
| **检索复杂度** | 只是简单 semantic search，还是要复杂过滤、混合检索、多租户 |
| **团队运维能力** | 能不能接受引入一套新组件 |
| **现有基础设施** | 已经有 PostgreSQL、Kubernetes、对象存储等吗 |

只要这 4 个问题答清楚，选型通常不会太偏。

## pgvector 什么时候最合适

如果团队已经有 PostgreSQL，而且当前 RAG 规模还不算特别大，`pgvector` 往往是最稳的起点。

适合：

- 早期项目
- 中小规模知识库
- 想把业务数据和向量索引放在一个系统里
- 团队希望先少引一个组件

优点：

- 学习和迁移成本低
- 可以直接复用 Postgres 的权限、备份、监控和运维体系
- 做 metadata filter、租户隔离、业务 join 很顺

代价：

- 不是为超大规模向量检索原生设计
- 当数据量和吞吐继续往上走时，压力会更明显

## Milvus 什么时候更值

`Milvus` 更像专门面向大规模向量检索场景的基础设施。

适合：

- 向量规模明显更大
- 检索吞吐和性能要求更高
- 团队愿意接受更独立的检索系统

优点：

- 专门针对向量检索优化
- 在大规模场景里更容易把性能做起来
- 生态里经常被拿来做专业检索底座

代价：

- 运维复杂度更高
- 系统架构更重
- 对小团队来说，早期成本未必划算

## Weaviate 什么时候有优势

`Weaviate` 的优势通常不只是“存向量”，还包括更完整的数据组织和语义检索生态体验。

适合：

- 希望快速搭比较完整的语义检索 / 知识系统
- 偏向“平台化能力”而不是只做最小向量存储
- 愿意接受额外组件和一定学习成本

优点：

- 检索体验和生态整合度不错
- 对某些知识图谱化、对象化组织场景更友好

代价：

- 对只想做一个简单 RAG 的团队，可能略重
- 运维和理解成本高于 pgvector

## Qdrant 什么时候很顺手

`Qdrant` 往往给人的感受是：专注、清晰、工程上比较顺手。

适合：

- 想用专门向量数据库，但又不想一上来就上特别重的系统
- 重视 metadata filter 和工程使用体验
- 想在复杂度和能力之间找一个中间点

优点：

- 面向向量检索场景较专注
- 工程体验通常比较直接
- 对中型项目很友好

代价：

- 毕竟还是额外引入一个新系统
- 如果你已经强依赖 PostgreSQL，全链路一体化上不如 pgvector 自然

## 一个够用的选型建议

如果你想先要一个不容易错的默认建议，可以这样看：

| 当前阶段 | 更稳的默认选择 |
| --- | --- |
| **已有 Postgres，项目刚起步** | `pgvector` |
| **检索已成核心能力，中等规模增长明显** | `Qdrant` 或 `Weaviate` |
| **向量规模和吞吐都很高，愿意投入专门基础设施** | `Milvus` |

一句话版：

- **先能跑，再谈极限**
- **先看团队现有基础设施，再看“理论最优”**

## 迁移时最容易踩的坑

- 一开始数据量不大，却过早引入重型系统
- 明明已经有 PostgreSQL，却把业务数据和向量系统拆得很散
- 只看检索延迟，不看备份、权限、监控、运维和团队认知成本
- 忽略 metadata filter、多租户和业务 join 需求
- 没有把 RAG 问题拆开，误把“召回差”全怪到数据库选型上
