## 初级阶段书单

目标：建立扎实基本功。

### 《图解 HTTP》

- 适合阶段：初级
- 推荐理由：补网络与 Web 通信基础，适合作为 HTTP 入门
- 建议阅读顺序：前置阅读
- 是否必须精读：建议通读

### 《MySQL 必知必会》或《高性能 MySQL》

- 适合阶段：初级
- 推荐理由：前者适合入门，后者适合进阶理解数据库性能问题
- 建议阅读顺序：HTTP 之后、Redis 之前
- 是否必须精读：入门书通读，进阶书按主题精读

### 《Redis 设计与实现》

- 适合阶段：初级
- 推荐理由：帮助理解缓存、数据结构、持久化和 Redis 内部机制
- 建议阅读顺序：数据库基础之后
- 是否必须精读：建议精读核心章节

### 《深入理解计算机系统》

- 适合阶段：初级
- 推荐理由：建立系统底层认知，理解程序、内存、IO、性能基础
- 建议阅读顺序：可贯穿整个初级阶段
- 是否必须精读：建议选重点章节精读

### 《UNIX/Linux 系统管理技术手册》或常见 Linux 实战资料

- 适合阶段：初级
- 推荐理由：补运维基础，便于排查进程、内存、磁盘、网络问题
- 建议阅读顺序：可与后端基础并行
- 是否必须精读：实践优先，按场景精读

### 《Head First 设计模式》或《设计模式》

- 适合阶段：初级
- 推荐理由：建立设计思维，理解常见模式及适用边界
- 建议阅读顺序：数据库和网络基础之后
- 是否必须精读：建议精读常用模式

## 中级阶段书单

目标：进入分布式与高并发。

### 《大型网站技术架构》

- 适合阶段：中级
- 推荐理由：很适合中文语境下理解架构演进
- 建议阅读顺序：中级阶段先读
- 是否必须精读：建议精读

### 《从 0 开始学架构》

- 适合阶段：中级
- 推荐理由：偏方法论，适合建立整体视角
- 建议阅读顺序：和《大型网站技术架构》并行
- 是否必须精读：建议通读

### 《高并发系统设计 40 问》

- 适合阶段：中级
- 推荐理由：适合结合真实业务场景理解高并发问题
- 建议阅读顺序：掌握缓存和消息队列基础后
- 是否必须精读：建议按高频场景精读

### 《数据密集型应用系统设计》

- 适合阶段：中级
- 推荐理由：非常重要，适合建立分布式、存储、数据系统整体视角
- 建议阅读顺序：中级阶段重点阅读
- 是否必须精读：建议精读

### 《Kafka 权威指南》

- 适合阶段：中级
- 推荐理由：帮助理解消息系统设计与实践
- 建议阅读顺序：在消息队列基础之后
- 是否必须精读：建议精读核心章节

### 《Elasticsearch 实战》

- 适合阶段：中级
- 推荐理由：补充搜索、索引、检索系统能力
- 建议阅读顺序：可作为专题补充阅读
- 是否必须精读：按业务需要精读

### 《凤凰项目》与《独角兽项目》

- 适合阶段：中级
- 推荐理由：帮助理解工程协作、交付流程和组织效率问题
- 建议阅读顺序：与工程化主题并行
- 是否必须精读：建议通读

## 高级阶段书单

目标：从技术点走向架构方法论。

### 《软件架构设计：大型网站技术架构与业务架构融合之道》或同类架构方法书

- 适合阶段：高级
- 推荐理由：帮助把技术架构和业务架构放到同一视角里理解
- 建议阅读顺序：高级阶段前置
- 是否必须精读：建议精读

### 《实现领域驱动设计》

- 适合阶段：高级
- 推荐理由：适合进入 DDD 实践层
- 建议阅读顺序：高级阶段重点阅读
- 是否必须精读：建议精读

### 《领域驱动设计》

- 适合阶段：高级
- 推荐理由：经典但偏厚，更适合中高级之后深入阅读
- 建议阅读顺序：在《实现领域驱动设计》之后
- 是否必须精读：建议选重点章节精读

### 《企业集成模式》

- 适合阶段：高级
- 推荐理由：理解系统集成、消息模式和边界协作
- 建议阅读顺序：DDD 与分布式系统之后
- 是否必须精读：建议按主题精读

### 《Clean Architecture》

- 适合阶段：高级
- 推荐理由：帮助理解架构分层和边界控制
- 建议阅读顺序：方法论阶段并行阅读
- 是否必须精读：建议精读

### 《Building Evolutionary Architectures》

- 适合阶段：高级
- 推荐理由：理解演进式架构和渐进治理思路
- 建议阅读顺序：高级阶段中后段
- 是否必须精读：建议精读

### 《Site Reliability Engineering》

- 适合阶段：高级
- 推荐理由：稳定性治理非常值得看，适合建立工程与可靠性视角
- 建议阅读顺序：高级阶段中后段
- 是否必须精读：建议精读核心章节

## 简版阅读顺序

1. 《图解 HTTP》
2. 《MySQL 必知必会》
3. 《Head First 设计模式》
4. 《大型网站技术架构》
5. 《数据密集型应用系统设计》
6. 《实现领域驱动设计》
7. 《Site Reliability Engineering》

- 适合阶段
- 推荐理由
- 建议阅读顺序
- 是否必须精读
