# 架构面试准备清单

## 适合人群

- 正在准备架构师、技术负责人、高级后端面试的工程师
- 想把零散知识整理成复习资料库的人
- 需要按高频模块系统复盘的人

## 学习目标

- 按高频面试模块建立复习范围
- 明确每个模块该看什么、练什么、怎么答
- 沉淀成适合长期复习的面试资料库

## 核心模块

### 计算机基础

- 操作系统：进程、线程、协程、内存、IO、多路复用
- 计算机网络：TCP/IP、HTTP/HTTPS、长连接、负载均衡
- 数据结构与算法：常见结构、复杂度分析、基础算法题

### 数据库

- MySQL：索引、事务、锁、MVCC、SQL 优化、分库分表
- Redis：数据结构、缓存击穿/穿透/雪崩、持久化、主从/哨兵/集群

### 分布式与高并发

- CAP、BASE、一致性
- MQ：Kafka / RabbitMQ 的使用场景、顺序、重复、积压、可靠性
- 分布式事务：2PC、TCC、消息最终一致性
- 限流、熔断、降级、幂等、重试

### 架构设计

- 高可用、高性能、高并发、可扩展
- 微服务拆分原则
- 服务治理、注册发现、配置中心、API 网关
- 灰度发布、容灾、多机房、多活

### 工程与治理

- CI/CD
- 监控、日志、链路追踪
- 稳定性治理
- 故障应急与复盘

### 业务设计

- DDD
- 用户、订单、支付、库存这些经典系统设计题
- 如何从业务增长角度做架构演进

## 推荐资料组合

书籍：

- 《大型网站技术架构》
- 《数据密集型应用系统设计》
- 《实现领域驱动设计》
- 《图解 HTTP》
- 《高性能 MySQL》

视频或专栏：

- 极客时间的架构、MySQL、Redis、Kafka、分布式系统专栏
- 各大厂架构面试经验分享
- 系统设计类公开视频课程

练习方式：

- 按专题整理自己的答案
- 每个专题写“原理 + 业务场景 + 常见问题 + 解决方案”
- 对高频题反复输出口述版答案

## 高频面试题方向

- MySQL 索引为什么能加速查询
- Redis 缓存雪崩怎么处理
- 消息重复消费怎么保证幂等
- 如何设计一个秒杀系统
- 如何设计订单系统
- 分库分表后如何解决分页和事务问题
- 系统怎么做高可用
- 如何做接口限流与熔断
- 如何设计一个亿级流量系统

## 最推荐的整理方式

每个主题统一成 4 列：

- 知识点
- 核心原理
- 典型面试题
- 标准回答思路

最终会形成一份更适合复习的架构面试资料库。
