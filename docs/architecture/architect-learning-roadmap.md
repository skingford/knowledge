## 适合人群

- 已有后端开发经验，想从“会实现”升级到“会设计”
- 需要准备架构岗、技术负责人或高级工程师能力自检
- 希望建立系统设计、稳定性治理、技术规划能力

## 学习目标

- 能看懂并评估一个系统的整体设计
- 能根据业务规模做架构演进
- 能从性能、稳定性、成本三个维度评估技术方案

## 学习路线

### 初级阶段：建立系统基础认知

学习目标：理解一个系统如何运行，而不只是会写业务功能。

重点内容：

- 计算机基础：操作系统、网络、数据结构、算法基础
- 数据库：MySQL、索引、事务、锁、SQL 优化
- 后端基础：接口设计、认证鉴权、缓存、消息队列入门
- 设计能力：面向对象设计、设计模式、模块拆分
- Linux：常用命令、进程、内存、磁盘、网络排查

阶段目标：

- 能独立设计一个中小型系统
- 能看懂典型 Web 架构
- 能分析慢 SQL、接口慢、服务异常等常见问题

推荐实践：

- 做一个完整后台系统
- 引入 Redis、MQ、日志、监控
- 输出系统架构图和部署图

### 中级阶段：进入系统设计与分布式

学习目标：能设计可扩展、可维护、抗压的系统。

重点内容：

- 高并发：缓存、限流、降级、熔断、异步削峰
- 高可用：主从、集群、故障转移、容灾、重试机制
- 分布式系统：CAP、BASE、一致性、分布式事务
- 微服务：服务拆分、注册发现、配置中心、网关
- 中间件：Redis、Kafka/RabbitMQ、Elasticsearch、Nginx
- 可观测性：日志、指标、链路追踪、告警

阶段目标：

- 能主导一个中大型系统模块设计
- 能根据业务增长进行架构演进
- 能处理性能瓶颈和稳定性问题

推荐实践：

- 设计秒杀系统、订单系统或支付系统
- 做压测并分析瓶颈
- 输出容量评估、监控方案、容灾方案

### 高级阶段：走向架构治理与业务抽象

学习目标：从单点技术强，升级到全局设计强。

重点内容：

- 架构方法论：DDD、事件驱动、分层架构、整洁架构
- 业务架构：领域划分、边界上下文、系统拆分与整合
- 平台化：中台、共享能力平台、统一技术底座
- 治理体系：代码规范、发布流程、变更管理、SLA/SLO、风险治理
- 云原生：Docker、Kubernetes、服务网格、弹性伸缩
- 安全架构：权限体系、审计、数据安全、攻防意识
- 成本与效率：资源成本、研发效率、稳定性投入产出比

阶段目标：

- 能从业务目标反推技术方案
- 能主导跨团队架构设计
- 能做系统演进规划，而不是只做局部优化

推荐实践：

- 主导老系统拆分与迁移方案
- 设计多团队共享平台
- 输出技术规划、架构评审模板、风险清单

## 核心知识地图

### 核心基础

- 操作系统：进程、线程、协程、内存、IO、多路复用
- 计算机网络：TCP/IP、HTTP/HTTPS、长连接、负载均衡
- 数据结构与算法：常见结构、复杂度分析、基础算法题
- 设计模式：常见模式、适用场景、设计权衡

### 数据架构

- MySQL：索引、事务、锁、MVCC、SQL 优化、分库分表
- Redis：数据结构、缓存击穿/穿透/雪崩、持久化、主从/哨兵/集群
- 消息队列：Kafka、RabbitMQ 的使用场景、顺序、重复、积压、可靠性

### 分布式与微服务

- CAP、BASE、一致性
- 分布式事务：2PC、TCC、消息最终一致性
- 微服务拆分原则
- 服务治理、注册发现、配置中心、API 网关

### 稳定性与工程治理

- 限流、熔断、降级、幂等、重试
- CI/CD
- 监控、日志、链路追踪
- 灰度发布、容灾、多机房、多活
- 故障应急与复盘
- 测试、发布、安全治理

### 业务与方法论

- DDD
- 领域建模
- 事件驱动
- 分层架构、整洁架构
- 系统拆分
- 从业务增长角度做架构演进

## 推荐书单

### 初级阶段

- 《图解 HTTP》：补网络与 Web 通信基础
- 《MySQL 必知必会》或《高性能 MySQL》：先入门再进阶
- 《Redis 设计与实现》：理解缓存和数据结构
- 《深入理解计算机系统》：建立系统底层认知
- 《UNIX/Linux 系统管理技术手册》或同类 Linux 实战资料：补运维基础
- 《Head First 设计模式》或《设计模式》：建立设计思维

### 中级阶段

- 《大型网站技术架构》：适合中文语境下入门架构演进
- 《从 0 开始学架构》：偏方法论，适合建立整体视角
- 《高并发系统设计 40 问》：适合结合业务场景理解问题
- 《数据密集型应用系统设计》：建议重点阅读
- 《Kafka 权威指南》：理解消息系统
- 《Elasticsearch 实战》：补充搜索与检索能力
- 《凤凰项目》与《独角兽项目》：帮助理解工程协作与交付体系

### 高级阶段

- 《软件架构设计：大型网站技术架构与业务架构融合之道》或同类架构方法书
- 《实现领域驱动设计》：适合进入 DDD
- 《领域驱动设计》：经典但偏厚，适合中高级阶段深入阅读
- 《企业集成模式》：理解系统集成与消息模式
- 《Clean Architecture》：理解架构分层与边界控制
- 《Building Evolutionary Architectures》：理解演进式架构
- 《Site Reliability Engineering》：稳定性治理必读
