# 高并发系统设计清单

## 适合人群

- 需要设计高并发接口或活动系统的后端工程师
- 准备系统设计能力自检的人
- 需要把“会说概念”变成“会做方案”的开发者

## 学习目标

- 建立高并发系统设计的基础检查框架
- 能从流量、性能、稳定性、数据一致性几个维度审题
- 输出更完整的方案，而不是只讲缓存和 MQ

## 设计总清单

### 先确认业务约束

- 峰值流量多大
- 是否存在突发流量
- 是否允许排队、降级、限流
- 是否要求强一致
- 用户体验要求是什么

### 再确认系统容量

- QPS、TPS、并发连接数
- 数据量级和增长速度
- 缓存命中率预期
- 数据库、消息队列、下游服务瓶颈

### 最后确认保护策略

- 限流
- 熔断
- 降级
- 超时
- 重试
- 隔离

## 核心模块

### 接入层

- 负载均衡
- 网关限流
- 验签、鉴权、防刷

### 应用层

- 无状态扩容
- 热点隔离
- 异步化削峰
- 幂等控制

### 数据层

- 缓存优先
- 读写分离
- 分库分表
- 库存扣减策略

### 稳定性治理

- 监控与告警
- 链路追踪
- 容灾预案
- 故障演练

## 典型风险

- 热点 Key 打爆缓存
- 数据库被穿透或写爆
- 重复请求导致脏数据
- 消息堆积导致链路延迟放大
- 下游故障级联传染

## 能力自检回答结构

1. 先讲业务目标和约束
2. 再讲整体架构和关键链路
3. 再讲性能与容量设计
4. 再讲一致性与幂等
5. 最后讲稳定性、监控和兜底方案

## 推荐实践项目

- 秒杀系统
- 抢券系统
- 高并发下单系统
- 活动报名系统
