## 基础与底层

### 1. Slice 的底层结构是什么？

底层是一个包含指针、长度、容量的结构体，分别指向底层数组、当前长度和容量。

### 2. Slice 扩容机制是怎样的？

追加元素超过容量时会扩容。小容量通常增长更快，大容量增长更平滑，具体比例和 Go 版本实现有关。

### 3. Map 为什么并发不安全？

因为读写过程中可能触发扩容和迁移，并发访问会破坏内部状态，所以原生 Map 不保证并发安全。

### 4. Channel 的底层原理是什么？

Channel 内部维护队列、发送等待队列、接收等待队列和锁，用于在 Goroutine 之间做安全通信与同步。

### 5. Interface 的底层结构是什么？

Interface 本质上包含类型信息和数据指针。空接口和非空接口在底层表示上有所不同。

### 6. Nil Interface 和 Nil 指针有什么区别？

Nil Interface 是类型和值都为空；如果 Interface 里装的是一个类型已知但值为 nil 的指针，它本身并不等于 nil。

### 7. Defer 的执行顺序是什么？

后定义的 `defer` 先执行，整体是后进先出。

### 8. Panic 和 Recover 的正确使用方式是什么？

`panic` 用于不可恢复错误，`recover` 只能在 `defer` 中捕获。正常业务错误不应该大量依赖 `panic`。

### 9. 什么是逃逸分析？

编译器判断对象应该分配在栈上还是堆上的过程。逃逸到堆会增加 GC 压力。

### 10. String 和 `[]byte` 转换要注意什么？

转换通常会涉及拷贝和分配，要关注性能影响，尤其是在高频路径中。

## 并发与调度

### 11. Goroutine 为什么轻量？

初始栈小、按需扩缩，调度由 Runtime 管理，不直接一一对应系统线程，所以创建和切换成本更低。

### 12. 什么是 GMP 模型？

G 是 Goroutine，M 是线程，P 是调度上下文。P 负责把 G 调度到 M 上执行。

### 13. Go 的抢占式调度是什么？

Runtime 会在合适时机中断长时间运行的 Goroutine，避免其长期占用 CPU，提升调度公平性。

### 14. Channel 和 Mutex 怎么选？

共享状态保护优先考虑 Mutex；任务和数据传递优先考虑 Channel，不要为了“优雅”滥用 Channel。

### 15. Select 有什么作用？

`select` 用于同时监听多个 Channel 操作，适合超时控制、多路收发和取消信号处理。

### 16. Context 有什么用？

用于传递取消信号、超时和少量请求级上下文数据，是请求链路控制和资源回收的重要机制。

### 17. 为什么 Context 不应该滥用 `WithValue`？

因为它会让依赖关系变隐式，降低可读性和可维护性。它只适合少量跨层协作信息。

### 18. 什么是 Goroutine 泄漏？

Goroutine 本该退出却长期不退出，通常由 Channel 阻塞、缺少取消逻辑、死循环重试等导致。

### 19. Atomic 和 Mutex 的区别是什么？

Atomic 适合简单原子变量操作，开销更小；Mutex 适合保护复杂共享状态。

### 20. `sync.Map` 适合什么场景？

适合读多写少、键集合动态变化的并发场景，不适合作为普通 Map 的默认替代品。

## 工程与性能

### 21. Go 的 GC 是如何工作的？

Go 使用并发垃圾回收，以降低停顿时间，但频繁分配和大对象仍会对延迟产生影响。

### 22. GC 为什么会影响延迟？

对象越多、分配越频繁、存活时间越长，GC 的扫描和回收成本越高，可能带来延迟抖动。

### 23. 如何减少内存分配？

减少临时对象、避免不必要拷贝、复用对象、控制字符串和切片分配、合理使用对象池。

### 24. `sync.Pool` 适合什么场景？

适合短生命周期、可复用对象的临时缓存，不适合拿来做长期资源池。

### 25. 如何用 Pprof 排查性能问题？

先确认问题类型，再看对应 Profile，例如 CPU、Heap、Goroutine、Mutex、Block，然后结合代码和请求场景验证。

### 26. 如何排查内存泄漏？

关注对象是否被长期持有，通过 Heap Profile、对象引用链和缓存生命周期分析定位问题。

### 27. 如何排查慢请求？

结合日志、Trace、Pprof、数据库慢查询、下游调用耗时，逐层定位瓶颈。

## 系统设计结合 Go

### 28. Go 服务如何做高并发优化？

从连接池、超时、限流、减少分配、锁优化、异步化和缓存几个维度同时考虑，而不是只看代码层面。

### 29. Go 在微服务架构中的优势是什么？

启动快、部署简单、并发模型清晰、资源占用相对可控，适合服务端和基础设施场景。

### 30. 用 Go 设计高并发接口时重点看什么？

重点看限流、超时、幂等、缓存、数据库压力、异步化、监控和故障兜底。
