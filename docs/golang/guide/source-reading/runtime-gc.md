# Go GC：垃圾回收源码精读

> 核心源码：`src/runtime/mgc.go`、`src/runtime/mgcmark.go`、`src/runtime/mgcsweep.go`

## GC 全景图

```
Go GC 三色标记-清扫（Tri-color Mark-Sweep）
══════════════════════════════════════════════════════════════════

  颜色语义：
  ┌────────────┬──────────────────────────────────────────────┐
  │  白色 White │ 未访问，GC 结束时仍为白 → 可回收            │
  │  灰色 Gray  │ 已发现但子对象未扫描完 → 工作队列中         │
  │  黑色 Black │ 自身和所有引用均已扫描 → 不可回收           │
  └────────────┴──────────────────────────────────────────────┘

  GC 阶段时序：
  ═══════════════════════════════════════════════════
  │STW│←────── 并发标记 ──────→│STW│← 并发清扫 →│
  │   │                        │   │             │
  │初│ 所有 goroutine 并发运行  │终│ 并发清扫   │
  │始│ GC 工作线程同时标记对象  │止│ sweep span │
  └──┴────────────────────────┴───┴─────────────┘
  GC 开始     并发标记阶段      标记结束     清扫阶段
  (STW ~1ms)                    (STW ~1ms)  (与用户并发)

══════════════════════════════════════════════════════════════════
```

## 一、GC 触发条件

```
GC 触发时机
══════════════════════════════════════════════════════

  1. 堆内存增长触发（最常见）
     ├── 触发条件：当前堆大小 ≥ 上次 GC 后堆大小 × (1 + GOGC/100)
     └── GOGC=100（默认）→ 堆翻倍时触发

  2. 定时触发
     └── sysmon 每隔 2 分钟强制触发一次（防止长时间无 GC）

  3. 手动触发
     └── runtime.GC() 显式调用（测试/基准测试场景）

  4. 内存限制触发（Go 1.19+）
     └── runtime/debug.SetMemoryLimit(n) 设置上限，接近时触发

══════════════════════════════════════════════════════
```

## 二、标记阶段

```
并发标记流程
══════════════════════════════════════════════════════

  STW（Stop-The-World）开始
  ├── 暂停所有 goroutine
  ├── 开启写屏障（Write Barrier）
  ├── 扫描根集合（全局变量、各 G 的栈）→ 加入灰色队列
  └── 恢复所有 goroutine（STW 结束，约 <1ms）
       │
       ▼
  并发标记阶段（GC goroutine 与用户 goroutine 并发）
  ├── markroot()：扫描根对象
  ├── gcDrain()：从灰色队列取对象 → 扫描引用 → 涂黑
  └── gcAssistAlloc：用户 goroutine 分配内存时辅助标记
       │
       ▼
  STW（Stop-The-World）结束标记
  ├── 处理最后一批灰色对象
  ├── 关闭写屏障
  └── 恢复所有 goroutine

══════════════════════════════════════════════════════
```

### 写屏障（Write Barrier）

```
写屏障解决并发标记的漏标问题
══════════════════════════════════════════════════════

  漏标场景（无写屏障时）：
  ├── A（黑）→ B（黑，已扫描）
  ├── 用户代码：A 删除对 C 的引用，B 新增对 C 的引用
  └── GC 不知道 C 已被 B 引用 → C 被错误回收 ❌

  混合写屏障（Go 1.14+，Hybrid Write Barrier）：
  ├── 写操作时：将被覆盖的旧值和新值都标为灰色
  └── 栈上对象不需要写屏障（STW 时扫描）

  伪代码：
  // slot = ptr 触发写屏障
  writePointer(slot *unsafe.Pointer, ptr unsafe.Pointer) {
      shade(*slot) // 旧值变灰
      shade(ptr)   // 新值变灰
      *slot = ptr
  }

══════════════════════════════════════════════════════
```

## 三、清扫阶段

```
并发清扫（gcSweep）
══════════════════════════════════════════════════════

  标记结束后异步执行（与用户代码并发）

  span（内存页组）状态扫描：
  ├── 检查 span 中每个对象的 GC mark bit
  ├── 白色对象 → 归还内存（加入 mcentral freelist）
  └── 黑色对象 → 保留，清除 mark bit（为下次 GC 准备）

  延迟清扫（lazy sweep）：
  └── 分配内存时顺便清扫相关 span（分摊清扫成本）

══════════════════════════════════════════════════════
```

## 四、GC 调优参数

```
关键参数一览
══════════════════════════════════════════════════════════════════

  GOGC（环境变量 / debug.SetGCPercent）
  ├── 默认：100（堆翻倍触发）
  ├── 调大（如200）：减少 GC 频率，增大内存占用
  ├── 调小（如50）：增加 GC 频率，降低内存峰值
  └── GOGC=off 或 -1：禁用自动 GC

  GOMEMLIMIT（Go 1.19+，环境变量 / debug.SetMemoryLimit）
  ├── 设置内存上限，GC 会更积极防止 OOM
  └── 推荐：替代 GOGC 小值的方案，更可预测

  GC 辅助（gcAssist）
  └── 分配大量内存的 goroutine 会被强制参与标记

══════════════════════════════════════════════════════════════════

  调优建议：
  ├── 高吞吐低延迟：GOMEMLIMIT=物理内存80% + GOGC=off
  ├── 内存敏感：保持 GOGC=100 + 减少堆分配
  └── 观测 GC：GODEBUG=gctrace=1
```

## 五、代码示例

```go
import (
    "runtime"
    "runtime/debug"
)

func init() {
    // 方案1：限制内存，触发更激进 GC
    debug.SetMemoryLimit(512 << 20) // 512MB 上限

    // 方案2：降低 GC 触发频率（接受更高内存）
    debug.SetGCPercent(200)
}

// 查看 GC 统计
func printGCStats() {
    var s runtime.MemStats
    runtime.ReadMemStats(&s)
    fmt.Printf("HeapAlloc: %d MB\n", s.HeapAlloc>>20)
    fmt.Printf("NumGC: %d\n", s.NumGC)
    fmt.Printf("PauseTotalNs: %d ms\n", s.PauseTotalNs/1e6)
}
```

```bash
# 观测 GC 行为（不需要修改代码）
GODEBUG=gctrace=1 go run main.go

#   0.076+0.22/0.40/0+0.013 ms cpu, 4->4->2 MB, 5 MB goal, ...
#   ↑      ↑          ↑              ↑
# GC次数  时间        pause时间       堆大小变化
```

## 面试要点

| 问题 | 要点 |
|------|------|
| Go GC 算法是什么？ | 三色并发标记-清扫（Tri-color Concurrent Mark-Sweep） |
| STW 发生几次？ | 每次 GC 两次：标记开始（初始化）和标记结束 |
| 为什么需要写屏障？ | 防止并发标记时用户代码导致漏标（对象被错误回收） |
| GOGC 默认值？ | 100，即堆增长到上次 GC 后 2 倍时触发 |
| 如何减少 GC 压力？ | 减少堆分配（用 sync.Pool）、对象复用、避免小对象堆逃逸 |
| Go 1.14 GC 改进？ | 引入混合写屏障，消除了对栈的写屏障，减少 STW 时间 |
