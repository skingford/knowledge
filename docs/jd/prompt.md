# 面试模拟问答

这个文档的"参考回答"格式是一个面试模拟问答，每个 Q 由三部分组成：

1. 问题标题 — #### Q{n}: 面试问题
2. 参考回答 — 用 > blockquote 模拟候选人第一人称口述
3. 追问准备 — 列出面试官可能的追问及简短应答

生成这类内容的提示词参考：

你是一位资深技术面试辅导教练。我正在准备 [岗位名称] 的面试。

请根据以下 JD 要求和技术栈，生成面试模拟问答：

- 岗位：[内容分发平台 高级 Golang 开发工程师]
- 核心技术栈：[Go-Zero, gRPC, Docker, K8s, Redis, Kafka, CDN, EMQX]
- 经验要求：[5年以上]

## 输出格式要求

对每个问题，严格按以下结构输出：

### Q{序号}: {面试官的提问}

**参考回答：**

> {用第一人称、口语化风格回答。像在面试现场自然地阐述，而非照本宣科。
> 回答要有具体细节（框架名、配置项、数字指标），展现实际项目经验。
> 分 2-3 段，先讲核心做法，再补充关键实现细节。}

**追问准备：**

- **{追问问题1}？** {简短有力的回答，1-2 句话}
- **{追问问题2}？** {简短有力的回答，1-2 句话}

---

## 回答风格要求

1. **第一人称叙述** — 用"我们的做法是…"、"我负责…"、"核心是…"开头，不要用教科书式定义
2. **具体不空洞** — 每个回答必须包含具体技术名词、配置参数或数量指标（如"镜像从 1GB 降到
20MB"、"缓存命中率 95%+"）
3. **有主见** — 对比不同方案时要给出明确选择和理由，不要罗列"方案A可以、方案B也行"
4. **追问要犀利** — 追问应该是面试官听完回答后最可能追问的点，通常是边界情况、故障场景、替代方案

## 内容范围

请围绕以下主题各生成 3-5 个问题：

1. [Go-Zero 微服务架构与 gRPC]
2. [Docker / K8s / CI/CD]
3. [CDN / 缓存 / 内容分发]
4. [消息队列与异步处理]
5. [系统设计与故障处理]

关键点是告诉 AI：

- 用 > blockquote 格式包裹参考回答（这让它在 VitePress 中渲染为引用样式）
- 用第一人称口语风格，而非百科式解释
- 每个回答要有具体数字和技术细节
- 追问用 **加粗问题？** 格式，后面跟简短回答
