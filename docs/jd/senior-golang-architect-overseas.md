# 高级 Golang 开发工程师 / 架构师 JD

## 职位名称

高级 Golang 开发工程师 / 架构师

## 岗位定位

这是一个偏高级服务端负责人画像的岗位，核心不是单纯写 Go 业务代码，而是同时覆盖：

- 海外项目服务端迭代
- 架构优化与稳定性治理
- 疑难问题排查与性能瓶颈处理
- 技术方案质量把控
- 服务端团队日常管理

## 工作职责

- 负责海外项目的服务端迭代
- 架构优化，保证服务稳定性
- 新技术探索并为技术 / 业务赋能
- 解决项目中的疑难杂症与性能瓶颈
- 负责服务端人员的日常管理
- 把控服务端技术方案质量

## 任职要求

### 硬性要求

- 全日制本科，计算机相关专业
- 5 年以上后台开发经验
- 熟悉数据结构、算法，编程风格良好
- 熟练掌握 Golang，了解语言底层细节
- 熟悉 TCP / UDP、HTTP / HTTPS、多进程 / 线程、Socket 编程
- 熟练使用 Redis、MySQL、MQ，了解底层原理
- 良好的问题解决能力，能承受压力
- 良好的团队协作、沟通表达能力

### 加分项

- 有社交类或 C 端产品开发经验
- 有 Clean Code 实践
- 能快速学习新领域 / 新技术
- 擅长用 AI 提高效率
- 有技术博客、GitHub 项目或技术贡献
- 有基础架构经验
- 对技术有追求
- 有服务端管理经验更佳

## 从 JD 映射出的能力重点

基于这份 JD，可以把岗位真正看重的能力归成几类：

### 1. Go 服务端深度

- 不只是会写业务逻辑，还要理解语言底层细节
- 需要能处理性能、并发、网络、运行时相关问题

### 2. 基础设施与中间件深度

- Redis、MySQL、MQ 不只是会用，还要知道底层原理和故障边界
- 需要能支撑高并发、稳定性和线上问题排查

### 3. 稳定性与架构能力

- 能做架构优化
- 能解决复杂故障和性能瓶颈
- 能把技术方案落到稳定、可维护的服务端系统

### 4. 业务理解与结果导向

- 海外项目和 C 端 / 社交类经验，说明岗位看重复杂业务场景
- 不是纯基础平台岗，而是要能把技术能力转成业务结果

### 5. 团队影响力

- 需要日常管理服务端人员
- 需要把控技术方案质量
- 说明这是偏高级骨干或小团队负责人岗位
