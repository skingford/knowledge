# IP 与路由

## 适合谁看

- 知道 TCP、HTTP 但说不清"包到底怎么从 A 机器走到 B 机器"的人
- 线上排障时遇到路由不通、跨网段访问失败，想快速定位问题层的人
- 做系统设计或基础设施时，需要理解 IP 和路由在整条链路里的位置的人

## 学习目标

- 知道 IP 层在解决什么问题
- 理解源 IP、目标 IP、子网掩码、路由表、NAT、TTL 各自的角色
- 能区分 IP 层和传输层（TCP/UDP）各自负责什么

## IP 层解决什么问题

IP 层解决的核心问题是：

- 数据包往哪发
- 下一跳是谁
- 如何跨网段到达目标机器

## 必须掌握的关键词

- 源 IP / 目标 IP
- 子网掩码
- 默认网关
- 路由表
- NAT
- TTL

## IP 和 TCP 的关系

一个很实用的理解是：

- TCP 更像"连接和传输规则"
- IP 更像"把包送到目的地的寻路系统"

所以如果根本路由不到，TCP 和 HTTP 再正确也没用。
