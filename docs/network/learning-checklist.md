# 网络学习 Checklist

这页按实战导向整理，不走纯考试路线。目标不是把网络知识背成一堆定义，而是让你能把一次请求的完整链路讲顺，能写基础网络服务，也能处理常见的 `502`、`504`、超时、连接失败和 TLS 报错。

## 适合谁看

- 做 Go 后端，希望把网络知识补成可用于开发、联调和线上排障能力的人
- 已经零散学过 HTTP、TCP、DNS，但还没有一条稳定学习主线的人
- 想把协议原理、Linux 命令、反向代理、Go 网络编程串成一套体系的人

## 怎么用这页

- 不要只打勾不动手；每个阶段至少完成一轮实操检查
- 第一轮先把完整请求链路建立起来，不要求一开始就把每个协议细节啃到最深
- Go 网络编程不要等网络全学完才开始，建议从 HTTP / TCP 阶段就同步写 demo

## 0. 学习目标先定好

- [ ] 能讲清“浏览器访问一个网址发生了什么”
- [ ] 能看懂常见网络报错
- [ ] 能排查 DNS / TCP / TLS / HTTP 问题
- [ ] 能用 Go 写基本网络服务
- [ ] 能处理线上常见 `502` / `504` / 超时 / 连接失败

## 1. 网络整体认知

这一阶段先建立地图，不钻细节。

- [ ] 知道 OSI 七层模型
- [ ] 知道 TCP/IP 四层模型
- [ ] 能说出每层大概负责什么
- [ ] 理解 IP、端口、域名分别是什么
- [ ] 能解释“一个请求从客户端到服务端大致经历了什么”

你学完这部分，至少能回答：

- [ ] 为什么网络要分层
- [ ] 为什么访问网站要先解析域名
- [ ] 为什么服务除了 IP 还要端口

## 2. HTTP / HTTPS 基础

这是最优先的。做后端的人，第一轮先把 HTTP、HTTPS、状态码和常见头部讲顺。

### HTTP

- [ ] 理解请求行、请求头、请求体
- [ ] 理解响应行、响应头、响应体
- [ ] 掌握 `GET` / `POST` / `PUT` / `DELETE`
- [ ] 掌握常见状态码：`200` / `301` / `302` / `400` / `401` / `403` / `404` / `500` / `502` / `504`
- [ ] 理解 Cookie / Session
- [ ] 理解 Keep-Alive
- [ ] 理解缓存基础
- [ ] 理解 CORS 基础
- [ ] 知道 HTTP/1.1、HTTP/2、HTTP/3 大概分别解决什么问题

### HTTPS

- [ ] 知道 HTTPS = HTTP + TLS
- [ ] 知道证书是干什么的
- [ ] 知道 CA 是什么
- [ ] 知道为什么 HTTPS 比 HTTP 安全

实操检查：

- [ ] 用 `curl -v https://example.com` 看过一次完整响应
- [ ] 用 `curl -I https://example.com` 看过响应头
- [ ] 能看懂常见请求头
- [ ] 能解释为什么有的网站会跳转到 HTTPS

## 3. DNS

很多线上问题其实不是服务挂了，而是解析错了、缓存没刷新、权威记录还没同步。

- [ ] 理解域名解析流程
- [ ] 知道递归 DNS 和权威 DNS
- [ ] 知道本地缓存、浏览器缓存、系统缓存
- [ ] 认识常见记录：`A` / `AAAA` / `CNAME` / `MX` / `TXT`
- [ ] 理解 TTL 是什么
- [ ] 知道为什么改了解析不会立刻生效

实操检查：

- [ ] 会用 `dig`
- [ ] 会用 `nslookup`
- [ ] 查过一个域名的 `A` 记录
- [ ] 查过一个域名的 `CNAME`
- [ ] 能判断“这是解析问题还是服务问题”

## 4. TCP / UDP

这是核心中的核心。后面很多超时、重传、连接堆积、代理报错，最终都要回到这一层理解。

### TCP

- [ ] 理解三次握手
- [ ] 理解四次挥手
- [ ] 知道 `TIME_WAIT` 是什么
- [ ] 理解序号和确认应答
- [ ] 理解重传机制
- [ ] 理解滑动窗口
- [ ] 理解流量控制
- [ ] 对拥塞控制有基本概念

### UDP

- [ ] 知道 UDP 是无连接的
- [ ] 知道 UDP 不保证可靠到达
- [ ] 知道 UDP 为什么更轻
- [ ] 知道 UDP 适合实时通信、音视频、DNS 等场景

### 端口

- [ ] 理解端口是什么
- [ ] 知道为什么是 `IP + 端口` 才能找到进程
- [ ] 记住常见端口：`80` / `443` / `22` / `3306` / `6379`

实操检查：

- [ ] 能解释为什么 TCP 要三次握手
- [ ] 能解释为什么服务退出后可能还有 `TIME_WAIT`
- [ ] 能区分“连接超时”和“连接被拒绝”

## 5. IP / 路由 / NAT / ARP

这部分决定你能不能回答“包为什么到不了”和“为什么内网服务从公网访问不到”。

- [ ] 理解 IPv4 基本结构
- [ ] 知道公网 IP 和私网 IP 的区别
- [ ] 理解子网掩码的大概作用
- [ ] 理解网关是什么
- [ ] 理解路由是什么
- [ ] 理解 NAT 是什么
- [ ] 理解 ARP 是什么
- [ ] 知道同网段通信和跨网段通信区别
- [ ] 知道 `ping` 大概是怎么工作的
- [ ] 知道 `traceroute` 在看什么

实操检查：

- [ ] 看过本机 IP 配置
- [ ] 知道默认网关是谁
- [ ] 用过 `ping`
- [ ] 用过 `traceroute` 或 `tracepath`
- [ ] 能解释“为什么内网机器不能直接被公网访问”

## 6. TLS / 安全基础

不用一上来学到特别深，但必须把“证书、信任链、域名校验、握手阶段”这些基本逻辑建立起来。

- [ ] 理解对称加密和非对称加密区别
- [ ] 理解数字证书作用
- [ ] 理解签名和摘要的大概意义
- [ ] 理解 TLS 握手的大致流程
- [ ] 知道中间人攻击是什么
- [ ] 知道证书过期会发生什么
- [ ] 知道域名不匹配为什么会报错

额外基础安全概念：

- [ ] 知道 CSRF 是什么
- [ ] 知道 XSS 是什么
- [ ] 知道 Token / JWT 的基础作用

## 7. Linux 网络排障工具

这一块非常重要，必须会。你排线上问题时，第一轮通常不是改代码，而是先拿命令收集证据。

必会命令：

- [ ] `ping`
- [ ] `traceroute` / `tracepath`
- [ ] `dig`
- [ ] `nslookup`
- [ ] `curl`
- [ ] `ss`
- [ ] `netstat`
- [ ] `lsof -i`
- [ ] `nc`
- [ ] `telnet`
- [ ] `tcpdump`
- [ ] `openssl s_client`

建议顺手会：

- [ ] `ip addr`
- [ ] `ip route`
- [ ] `mtr`

你要能做到：

- [ ] 查某个端口是否在监听
- [ ] 查某个服务是否能连通
- [ ] 查某个域名是否解析正确
- [ ] 查某个连接是超时、拒绝还是 TLS 错误
- [ ] 查某个进程占用了哪个端口

## 8. Web 基础设施

学到这里，你就开始像后端工程师而不是只会背协议的人了。

- [ ] 理解正向代理
- [ ] 理解反向代理
- [ ] 理解负载均衡
- [ ] 知道 Nginx 常见作用
- [ ] 知道 CDN 是干什么的
- [ ] 知道网关的作用
- [ ] 理解超时、重试、熔断的基本概念
- [ ] 知道健康检查是什么

实操检查：

- [ ] 自己配过一次 Nginx 反向代理
- [ ] 把请求转发到本地一个 Go 服务
- [ ] 故意制造一次 `502` 或 `504` 看现象
- [ ] 能解释 `502` 和 `504` 的常见区别

## 9. Go 网络编程

你做 Go，这部分要同步练。不要把 Go 网络编程看成独立专题，它本质上是在把前面的 HTTP、TCP、TLS、超时、连接复用落到代码里。

- [ ] 会用 `net/http` 写基础 HTTP 服务
- [ ] 会写简单路由
- [ ] 会设置 server timeout
- [ ] 会设置 client timeout
- [ ] 理解连接复用
- [ ] 理解 `context` 超时取消
- [ ] 会写一个简单 HTTP client
- [ ] 会做基本的 graceful shutdown
- [ ] 知道为什么 `http.Transport` 应该复用
- [ ] 知道为什么 `resp.Body` 必须关闭并尽量读完
- [ ] 了解 WebSocket 基础
- [ ] 了解 gRPC 基础

实操检查：

- [ ] 写一个 `/ping` 接口
- [ ] 写一个调用第三方 API 的 client
- [ ] 给 client 设置超时
- [ ] 故意把目标地址写错并观察报错
- [ ] 故意制造超时并定位原因

## 10. 抓包与观察

这一块会让你真正“看见网络”，知道报错到底卡在发包前、握手中还是应用层。

- [ ] 安装并会用 Wireshark 或 `tcpdump`
- [ ] 抓过一次 TCP 三次握手
- [ ] 抓过一次 HTTP 请求
- [ ] 看过一次 TLS 握手
- [ ] 会按 IP / 端口过滤
- [ ] 会按协议过滤
- [ ] 能从抓包里看出“有没有发出去、有没有回应”

## 11. 常见故障排查 Checklist

当“服务访问不了”时，按这个顺序查：

- [ ] 域名是否解析正确
- [ ] 目标 IP 是否可达
- [ ] 目标端口是否开放
- [ ] 服务是否真的在监听
- [ ] 是否被防火墙拦了
- [ ] 是否 TLS 证书有问题
- [ ] 是否代理 / Nginx 转发有问题
- [ ] 是否应用本身报错
- [ ] 是否请求超时
- [ ] 是否连接池或资源耗尽

## 12. 学完后你应该能独立解释的题目

这些题如果都能讲顺，网络入门基本就算扎实了。

- [ ] 浏览器访问一个网址完整发生了什么
- [ ] DNS 解析流程是什么
- [ ] TCP 为什么三次握手
- [ ] `TIME_WAIT` 为什么存在
- [ ] HTTPS 为什么安全
- [ ] 证书是怎么验证的
- [ ] 为什么能 `ping` 通但网页打不开
- [ ] 为什么会出现 `502` / `503` / `504`
- [ ] 反向代理和负载均衡有什么区别
- [ ] 超时、拒绝连接、DNS 失败的区别是什么

## 13. 推荐学习顺序

按这个打勾最合理：

第一阶段：

- [ ] 网络分层
- [ ] HTTP / HTTPS
- [ ] DNS

第二阶段：

- [ ] TCP / UDP
- [ ] IP / 路由 / NAT / ARP
- [ ] TLS

第三阶段：

- [ ] Linux 网络排障工具
- [ ] Nginx / 反向代理 / 负载均衡
- [ ] Go 网络编程

第四阶段：

- [ ] 抓包与观察
- [ ] 故障注入与排查演练
- [ ] 高频问题口述复盘

## 14. 建议最少做完的实战

如果你想把这条线真正学成工程能力，至少做完下面这些：

- [ ] 用 Go 写一个 `/ping` 服务，补上 `ReadTimeout`、`WriteTimeout`、`IdleTimeout` 和 graceful shutdown
- [ ] 写一个 HTTP client，分别观察 DNS 失败、连接被拒绝、TLS 证书错误、超时这几类报错
- [ ] 用 Nginx 反代本地 Go 服务，并故意制造一次 `502` 和一次 `504`
- [ ] 用 `tcpdump` 或 Wireshark 抓一次 TCP 三次握手和一次 HTTPS 请求
- [ ] 用 `dig`、`curl -v`、`curl --resolve`、`openssl s_client` 各排过一次域名或 TLS 问题
- [ ] 自己画一张“浏览器访问一个 URL 的完整链路图”，把 DNS、TCP、TLS、HTTP、代理、应用、下游依赖串起来
