## 先分清“地址转换”和“访问控制”

很多人第一次学到这里时，最容易把两件事混在一起：

- NAT：解决地址不够用、内外网地址映射和出入口统一的问题
- 防火墙 / ACL / 安全组：解决“哪些流量允许过，哪些不允许过”的问题

也就是说：

- NAT 更像“改地址”
- 防火墙更像“放不放行”

如果你不先把这两件事拆开，排障时就很容易误判。

## NAT 到底在解决什么问题

NAT 本质上做的是：

- 把一个地址转换成另一个地址

最常见场景是：

- 内网机器用私有 IP
- 出公网时统一转换成一个或少量公网 IP

### 为什么会需要 NAT

最核心原因有两个：

1. IPv4 地址不够用
2. 想把内部网络结构隐藏在一个统一出口之后

### 常见 NAT 类型

| 类型 | 作用 | 常见场景 |
| --- | --- | --- |
| SNAT | 改源地址 | 内网机器访问公网 |
| DNAT | 改目标地址 | 外网请求转发到内网服务 |
| PAT / 端口复用 | 地址转换时复用端口 | 家庭路由器、云上出口网关 |

### 最实用的理解方式

- 从内网主动访问外网，常见是 SNAT
- 从外网打进内网，常见是 DNAT 或端口映射

## 为什么内网能访问而外网不一定能访问

这是最经典的线上问题之一。

内网能访问，只说明：

- 服务在内网地址上监听正常
- 同网段或内网路由是通的

但外网要访问成功，通常还要额外满足：

- 有公网入口 IP 或域名
- 有 DNAT / 端口映射 / 负载均衡把请求转进来
- 防火墙 / 安全组允许目标端口
- 服务本身监听的地址和端口正确

也就是说：

- “内网通”不等于“公网通”

## 防火墙和安全组到底拦的是什么

防火墙、安全组、ACL 的共同点是：

- 它们都在控制流量是否允许通过

但所处位置可能不同：

- 主机防火墙：例如 `iptables` / `nftables`
- 云安全组：云厂商实例级访问控制
- 网络 ACL：子网级或网络边界级访问控制
- WAF：更偏七层 HTTP 攻击防护

### 一个很实用的排障心法

当你怀疑“服务不通”时，要分清问题可能卡在哪：

1. 地址是否正确
2. 路由是否可达
3. 端口是否监听
4. 防火墙 / 安全组是否放行
5. 上层代理是否正确转发

### 最常见的误解

很多人会说：

- 端口已经开了，为什么还不通

其实“开了”至少可能指三件不同的事：

- 应用进程在监听
- 主机防火墙放行了
- 云安全组也放行了

少任何一个，结果都可能是不通。

## 端口映射和内网穿透怎么理解

### 端口映射

端口映射的本质是：

- 把公网某个端口映射到内网某台机器的某个端口

例如：

- 公网 `1.2.3.4:443` -> 内网 `10.0.0.8:8443`

这类能力在家庭路由器、云 LB、反向代理和网关里都很常见。

### 内网穿透

内网穿透通常是为了解决：

- 目标机器没有公网入口
- 但你又想让外部流量访问到它

常见思路是：

- 让内网机器主动向一个公网可达的中转节点建一条连接
- 外部访问先到公网节点
- 再通过已建立的通道把流量转回内网机器

所以内网穿透的重点通常不是“神奇地把内网变公网”，而是：

- 借助一个公网中继，把原本外部打不进来的链路反向打通

## VPN 在解决什么问题

VPN 的核心价值是：

- 在公共网络上建立一条受保护的专用通信通道

最常见场景：

- 远程办公访问公司内网
- 分支机构接入总部网络
- 云上 VPC 之间安全互通

### VPN 最少要讲清什么

- 它不只是“翻墙”这个语境里的词
- 在企业网络里，它更常表示远程接入和私网互联
- 它重点解决的是安全传输、身份校验和网络隔离后的可达性问题

### 和内网穿透的区别

- VPN 更偏“建立一个受控、长期、成体系的私网通信通道”
- 内网穿透更偏“把某个原本外部够不到的入口临时或定向暴露出来”

## 线上最常见的几类误区

### 1. 服务监听在 `127.0.0.1`，却希望外部直接访问

如果服务只监听本机回环地址：

- 本机能访问
- 外部机器即使能路由到这台主机，也访问不到这个服务

### 2. 以为有公网 IP 就一定能从外部访问

还要同时确认：

- 安全组是否开放
- 主机防火墙是否放行
- 服务端口是否监听
- 入口代理是否配置正确

### 3. 以为 NAT 就等于安全

NAT 主要解决地址转换，不等于完整安全策略。

真正的安全边界还要看：

- 安全组
- ACL
- 主机防火墙
- 身份认证

### 4. 以为 VPN 建好了就代表所有网段都互通

很多时候问题仍可能出在：

- 路由没配
- 网段冲突
- DNS 没配
- 访问控制策略没同步放开
